# Contributing to NovelMap

Thanks for your interest in NovelMap.

## Ways to Help
- Clarify requirements (issues with examples)
- Improve docs and UX wording
- Prototype importers/exporters
- Build small, well-scoped features

## Principles
- Keep the core model stable: Entities, Relationships, Appearances
- Prefer boring, portable tech
- Document decisions (ADRs) as we make them

## Pull Requests
- Keep PRs small
- Explain "why" in the description
- Add/update docs when behavior changes
